# 🌡️ 嵌入式式智能测温系统 (STM32 Distributed Temp Sensor)

本项目是一个基于 STM32F103VBT6 的分布式智能测温系统，综合运用了片上外设资源，构建了一个具备数据采集、人机交互、分布式通信及掉电存储功能的完整嵌入式架构。

## ✨ 核心特性与技术亮点

### 1. 🔄 单代码库运行时主从切换 (State Machine)
告别传统开发中维护两套代码（主机/从机）的痛点。通过引入自定义 `SystemRole_TypeDef` 状态机，系统支持通过按键 (KEY3) 在运行时无缝进行 **Master** 与 **Slave** 角色的热切换，极大提升了分布式节点的灵活性与代码复用率。

### 2. 🛡️ 完整 ModBus RTU 协议栈与 CRC16 校验
放弃简单的串口透传，针对长距离 RS485 半双工总线实现了“主从问答”机制的 ModBus RTU 协议。所有数据包均包含 CRC16 循环冗余校验码，接收端通过严格的帧长度检测与校验计算，有效滤除总线干扰，保证工业级通信的可靠性。

### 3. 💾 EEPROM 差异化写入与寿命保护
针对 AT24C02 EEPROM 擦写寿命有限（约 100 万次）的物理特性，设计了“差异化写入”策略：
- **按需写入**：拒绝周期性盲写，仅在 NTC 采集到的温度发生实质性变化时触发写入。
- **物理延时**：在 I2C 硬件层驱动中加入 5ms 物理延时，确保芯片内部写周期完整闭环，实现掉电历史数据的精准恢复。

### 4. ⚡ 前台无阻塞动态显示扫描
在存在 I2C 读写和 RS485 通信（毫秒级阻塞）的复杂场景下，为了保证多位数码管显示不闪烁（无频闪、无鬼影），摒弃了传统的死循环延时，采用了**前台高频轮询**策略，将 `Display_Scan_Loop` 嵌入到所有可能的等待间隙中。

## 🛠️ 硬件架构与引脚分配

- **MCU**: STM32F103VBTx (LQFP100)
- **开发环境**: STM32CubeMX + STM32CubeIDE + HAL 库
- **关键外设映射**:
  - `ADC1_IN15` (ADC): NTC 热敏电阻电压采集（配合 Steinhart-Hart 公式计算）
  - `USART2` (9600bps): 绑定 RS485 收发芯片，控制脚为 `PE6`
  - `USART1` (115200bps): 供 PC 上位机串口调试助手可视化监控系统日志
  - `I2C1`: 板载 AT24C02 EEPROM 读写
  - `GPIO`: 数码管段选 (A-H) 与位选 (SEL0-2)、按键 (KEY1-3)
